  1. Diferencia entre los datos de entrenamiento y validacin:
    Los datos de entrenamiento son usados para ajustar el modelo, el modelo aprende los patrones de estos datos. Los datos de validacin, son utilizados para evaluar el rendimiento del modelo durante el proceso de entrenamiento, ajustando hiperparmetros y detectando sobre-entrenamiento. Los datos de validacin son para ver que tan bien generaliza el modelo a datos nunca vistos.

  2. Desafo en problemas de clasificacin con data no supervisada:
    El principal desafo es que no hay etiquetas pre-existentes, por lo que es difcil evaluar la calidad de las clasificaciones.

  3. Matriz de confusin:
    La matriz de confusin es una tabla que muestra las predicciones correctas e incorrectas para cada clase. Tiene cuatro valores: verdaderos positivos, los que eran positivo que el modelo identifico como positivo (TP), verdaderos negativos, los que eran negativos que el modelo identifico como negativos (TN), falsos positivos, los que eran negativos y el modelo los identifico como positivos (FP), y falsos negativos, los que eran positivos y el modelo identifico como negativos (FN). Sirve para evaluar la precisin del modelo

  4. Mtricas y sus frmulas:

    • Accuracy:
      Frmula: $ \text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN} $
      Interpretacin: Proporcion de predicciones que el modelo identifica correctamente, frente al total de predicciones.

    • Precision:
      Frmula: $ \text{Precision} = \frac{TP}{TP + FP} $
      Interpretacin: Muestra el porcentaje de cuantas predicciones positivas fueron correctas.

    • Recall:
      Frmula: $ \text{Recall} = \frac{TP}{TP + FN} $
      Interpretacin: Mide el porcentaje del modelo para encontrar todos los positivos verdaderos.

    • F1 score:

      • Frmula: $ \text{F1} = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}} $
      • Interpretacin: Es una mtrica que combina Precision y Recall utilizando su media armnica. El F1 score es til cuando se busca un equilibrio entre ambas mtricas y es especialmente relevante en casos donde hay un desbalance entre las clases, ya que toma en cuenta tanto los falsos positivos como los falsos negativos.

  5. Mtricas recomendadas por contexto:

    • Mantenimiento predictivo de fallas de maquinaria pesada:
      Recall. Es importante minimizar los falsos negativos, ya que pasar una falla podra resultar en altos costos o accidentes.

    • Deteccin de enfermedades altamente contagiosas:
      Recall. Es indispensable encontrar la mayor cantidad de casos posibles positivos. Para esto seria ms util el Recall, ya que con un mayor recall estariamos identificando en mayor cantidad positivos verdaderos, aunque esto nos lleve a tambien identificar ms falsos negativos.

    • Aprobacin de crditos de alto riesgo:
      Precision. Se quiere asegurar que las personas a las que se les aprueba un crdito sean aquellas que probablemente puedan pagarlo, minimizando los falsos positivos.

    • Deteccin de crmenes:
      F1 score. Es necesario un balance entre precision y recall para no acusar falsamente a personas inocentes (precision), pero tambin para detectar a la mayor cantidad de criminales posible (recall).

  6. Calibracin de modelos:
    Es el proceso de ajustar las probabilidades predichas por un modelo de clasificacin para que sea mejor en la realidad. Se usa principalmente para mejorar la interpretacin de las probabilidades, hacindolas ms confiables para la toma de decisiones.

Variables Numricas

Se dividen en dos grupos: aquellas que tienen outliers y aquellas que no.

Variables con outliers (withoutliers_features):

  • StandardScaler().
    • Justificacin: La estandarizacin asegura que los outliers se mantengan dentro de un rango controlado. Este mtodo es adecuado porque en algunos algoritmos (p. ej., SVM, regresin logstica), la presencia de outliers puede influir en gran medida los resultados si no se normalizan correctamente. Adems, para estos algoritmos, es importante que las caractersticas estn en la misma escala.

Variables sin outliers (normal_features):

  • MinMaxScaler().
    • Justificacin: El escalado MinMax convierte los valores a un rango especfico (por defecto de 0 a 1), manteniendo la relacin proporcional entre los valores. Esta tcnica es adecuada cuando no hay outliers, porque no distorsiona la distribucin de los datos y mantiene las diferencias relativas entre los jugadores, lo cual es til para ciertos algoritmos como redes neuronales y KNN.

Variables Categricas

Variables categricas nominales (categorical_features):

  • OneHotEncoder().
    • Justificacin: El OneHotEncoder crea una columna binaria (0 o 1) para cada categora nica en estas variables. Esto es esencial para variables nominales, ya que evita asignar un orden arbitrario entre las categoras, lo que podra inducir un sesgo en los modelos de machine learning.
